= Determine if a ransomware attack is real
:icons: font
:imagesdir: ../media/

[.lead]
When Autonomous Ransomware Protection (ARP) detects abnormal activity in a protected volume, it issues a warning and displays a list of suspected files or details of entropy spikes. You are responsible for evaluating this unusual activity to determine whether the activity is acceptable (false positive) or whether an attack seems malicious. 

include::../_include/arp-requires-manual-investigation-of-files.adoc[]

The following are examples of steps you might take to determine if a ransomware attack is happening.

IMPORTANT: The following guidelines are provided for informational purposes only to assist with your investigation. NetApp makes no representations or warranties regarding the accuracy or completeness of these guidelines. You are solely responsible for evaluating all alerts, investigating suspected files, and determining the appropriate response to any potential security threats in your environment.

.Analyze file extensions

Examine the file extensions of reported files. An obvious sign of ransomware is files that have unusual or random character combinations appended to their names. For example, a file called `document.docx` might become `document.docx.wcry`.

Known ransomware extensions include `.wcry`, `.locked`, `.akira`, `.zcrypt`, `.phobos`, and others. Research the extension online or with AI tools.

If the extension is not associated with ransomware, the alert is likely a false positive. If the extension is associated with ransomware, the likelihood of a real attack is greater.

NOTE: Some ransomware does not change file extensions. The absence of unusual extensions does not eliminate the possibility of a ransomware attack.

.Consider the timing of the alert

If the alert occurs within a few hours or days of enabling ARP, it is likely a false positive. If no alerts occurred for several days after enabling ARP and then the alert appears, the likelihood of a true attack is greater.

.Attempt to open the file

You can try to open flagged files with their associated applications.

If the file opens and content is normal, it is likely a false positive. If the file cannot be opened or content is unreadable, it might be encrypted by ransomware.

CAUTION: Opening suspected files carries risk. Ensure you follow your organization's security protocols when attempting to access potentially compromised files.

.Look for ransomware notes

Check for ransomware notes in affected directories (for example, `README.txt` or `DECRYPT_INSTRUCTIONS.html`).

.Analyze system and application behavior

If systems and applications are operating normally, the alert is likely a false positive. A sudden and unexplained spike in file system activity (reads, writes, and deletions), however, often indicates that ransomware is actively encrypting files in the background.
