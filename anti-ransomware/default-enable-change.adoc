= Opt out of ONTAP Autonomous Ransomware Protection default enablement
:toclevels: 1
:imagesdir: ../media/

[.lead]
Beginning with ONTAP 9.18.1 GA, Autonomous Ransomware Protection (ARP) is automatically enabled by default on all supported volumes after a 12-hour warmup period following an upgrade or new installation, provided an ARP license is installed. You can opt out of this default enablement during or after the 12-hour grace period using System Manager or the ONTAP CLI.

.Steps

You can use System Manager or the ONTAP CLI to manage ARP default enablement options.
[role="tabbed-block"]
====
.System Manager
--
. Select *Cluster*...
--
.CLI
--
* Check the default enablement status, use the following CLI command:
+
[source,cli]
----
security anti-ransomware auto-enable show
----
* Opt out of default enablement for existing and new volumes:
+
[source,cli]
----
security anti-ransomware auto-enable modify -default-existing-volume-state false -default-new-volume-state false
----
* Disable ARP only on volumes that were auto-enabled by this default:
+
[source,cli]
----
security anti-ransomware volume disable -volume * -auto-enabled-volumes-only true
----
--
====
