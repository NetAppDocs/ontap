= Manage and monitor ONTAP SVMs with System Manager
:toclevels: 1
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ./media/

[.lead]
Beginning with ONTAP 9.19.1, SVM administrators can monitor and manage their own SVMs using a dedicated System Manager dashboard.

.Before you begin
Your cluster administrator must provide you with the:

* Management IP address to the SVM
* Credentials (username and password) for the SVM System Manager dashboard
* Optionally, permissions to add Volumes on the SVM

.Steps

. Enter the management IP address of the SVM in a browser.
. Enter your username and password for the SVM administrator (vsadmin).
. Select *Sign in*. 
+
The ONTAP System Manager dashboard opens.
. You can view the following information about the SVM:
+
[cols="1,2" options="header"]
|===
| SVM information
| Description
| Protocols
| The protocols enabled on the SVM.
| Details
| NIS and LDAP information, language, and the retention period after a volume is deleted.
| Performance
| Latency, IOPS, and Throughput performance statistics by the Hour, Day, Week, Month, or Year for the selected protocol.
|===

. Optionally, you can download the data in a comma separated value (csv) file for further analysis. 
. Select *Storage > Volumes*.
+
The Volumes dashboard page opens.
+
[Note]
The cluster administrator can grant permission to add volumes on the SVM. If this permission was not granted, clicking the Add button will fail and an error message will display.
