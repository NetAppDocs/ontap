= Assign SVM administrator access with ONTAP System Manager

:toclevels: 1
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ./media/

[.lead]
Beginning in ONTAP 9.19.1, ONTAP cluster administrators can assign SVM administrator dashboard access to the SVM manager.

This is particularly useful in a tenanted ONTAP environment, where the cluster administrator can provide an SVM dashboard for each tenant.

Cluster administrators can allow the SVM administrator the ability to create volumes.

.Before you begin
Gather the following before you assign SVM administrator dashboard access:
* Obtain SVM administrator credentials and ensure the user is unlocked.
* Ensure there is a management IP address for the SVM.

*For new SVMs*:
When you create an SVM, enable the *Manage administrator account* checkbox and record the password for the *vsadmin* account. This ensures that the *vsadmin* account is unlocked.

Select the Add a network interface for storage VM management check box. Specify the following:
* Node
* IP address
* Subnet mask

*For existing SVMs*:

Verify if the SVM has a management network interface. If you do not already have a network interface, you can create one on the *Network interfaces > Overview* page, select *Add to create a network interface* and select the following:

* Select *Storage VM management* as the network interface type.
* Specify the *IP address* and the *Subnet mask* for the SVM management network interface.

When you create an SVM, ONTAP creates the *vsadmin* user account. Set the password and make sure the user account is unlocked.

.Steps

. Open the System Manager dashboard as the cluster administrator.
. Select *Cluster > Storage VMs*. 
. Select the SVM, open the *Settings* page and select *Security > Users and roles*. 
. Select image:icon_kebab.gif[more icon] for the *vsadmin* user and select:
.. Change password to update the *vsadmin* password.
.. Unlock to make sure it is usable.

== Assign SVM administrator access

To assign SVM administrator access, you need to provide the:

* SVM management IP address
* Volume provisioning access (optional, enable if you want the SVM administrator to create local volumes)

.Steps

To get the SVM IP address:

. Open the System Manager dashboard and sign in as the cluster administrator.
. Under Cluster, select *Storage VMs*.
. Under Network Interfaces, select the *Management* network interface and copy the *IP address*. 
+
Give the SVM administrator this IP address so they can log in and open the SVM administrator dashboard.

.Steps

To allow the SVM administrator to provision local volumes:

. Open the System Manager dashboard and sign in as the cluster administrator.
. Under Cluster, select *Storage VMs*.
. Select image:icon_kebab.gif[more icon] for the applicable SVM.
. Select *Edit*. 
. Under Resource Allocation, select the *Limit volume creation to preferred local tiers* checkbox. 
+
By default, all local tiers are included when you select the checkbox. You can remove specific tiers if necessary. If you remove all local tiers or leave the checkbox blank, SVM administrators cannot provision local volumes.
. Select *Save*.
