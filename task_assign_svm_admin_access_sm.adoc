= Assign SVM administrator access with ONTAP System Manager
:toclevels: 1
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ./media/

[.lead]
Beginning in ONTAP 9.19.1, as an ONTAP cluster administrator,, you can assign SVM administrator access to the person responsible for managing and monitoring the SVM. 

.Before you begin
Before you assign SVM administrator access, you need the SVM administrator credentials. 

*For new SVMs*:
When you create an SVM, enable the Manage administrator account checkbox and record the password for the vsadmin account. 

*For existing SVMs*:
•	If the vsadmin password is known, you can provide it to the SVM admin. 
•	If the vsadmin password is unknown, you will need to use the CLI. 

== Assign SVM administrator access
To assign SVM administrator access, you need to provide the:

* SVM IP address
* Volume provisioning access (optional, enable if you want the SVM administrator to create local volumes)

.Steps

To get the SVM IP address:

. Open the System Manager dashboard and sign in as the cluster administrator.
. Under Cluster, select *Storage VMs*.
. Under Network Interfaces, select the *Management* network interface and copy the *IP address*. 
+
Provide the SVM administrator with this IP address so they can log in and open the SVM administrator dashboard.

.Steps

To allow the SVM administrator to provision local volumes:

. Open the System Manager dashboard and sign in as the cluster administrator.
. Under Cluster, select *Storage VMs*.
. Select image:icon_kebab.gif[more icon] for the applicable SVM.
. Select *Edit*. 
. Under Resource Allocation, select the *Limit volume creation to preferred local tiers* checkbox. 
+
By default, all local tiers are included when you select the checkbox. You can remove specific tiers if necessary.
. Click *Save*.
