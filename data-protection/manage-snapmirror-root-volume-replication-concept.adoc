= Learn about ONTAP SnapMirror root volume replication
:icons: font
:imagesdir: ../media/

[.lead]
Every SVM in a NAS environment has a unique namespace. The SVM root volume is the entry point to this namespace hierarchy. For clusters consisting of two or more HA pairs, you should consider load-sharing mirrors (LSMs) of the SVM root volume to ensure the namespace remains accessible to clients in the event that both nodes of an HA pair fail. Load-sharing mirrors are not suitable for clusters consisting of a single HA pair and are not suitable for a MetroCluster environment.

* Beginning with ONTAP 9.1, load-sharing mirrors have been deprecated for data volumes and are only supported for SVM root volumes.

* SnapMirror load-sharing mirrors are only capable of supporting NAS (CIFS/NFSv3) clients. Load-sharing mirrors do not support NFSv4 clients or SAN client protocol connections (FC, FCoE, or iSCSI). However, NFSv4 and load-sharing mirrors can coexist in the same environment.

[NOTE]
LSMs have limited value in modern ONTAP cluster solutions because of the multiple hardware redundancies integrated into the hardware and cluster software. For most deployments, LSMs are not recommended. Instead, NetApp recommends using SVM disaster recovery as an alternative solution.
