= Enable or disable NFSv4.2 for ONTAP SVMs
:icons: font
:imagesdir: ../media/

[.lead]
Beginning with ONTAP 9.19.1, you can enable or disable NFSv4.2 by modifying the `-v4.2` option. This allows file access for clients using the NFSv4.2 protocol. 

The NFSv4.2 option allows for immediate protocol configuration changes, removing the need for cluster-wide boot arguments or disruptive node rebooks. 

NFSv4.2 support is managed independently of NFSv4.1 settings and global bootargs, allowing each SVM to advertise or suppress NFSv4.2 capabilities as needed. By default, NFSv4.2 is enabled when a new NFS service is created unless explicitly disabled by the administrator. 

The new configuration option ensures that NFSv4.2 settings persist across ONTAP upgrades, reverts, and SVM migrations, reducing the risk of misconfiguration and operational disruption. Only users with appropriate administrative privileges can modify the NFSv4.2 setting, and all changes are recorded in audit logs for compliance and troubleshooting purposes.

.Step

. Perform one of the following actions:
+
[cols="2*",options="header"]
|===
| If you want to...| Enter the following command...
a|
Enable NFSv4.2
a|
`vserver nfs modify -vserver vserver_name -v4.2 enabled`
a|
Disable NFSv4.2
a|
`vserver nfs modify -vserver vserver_name -v4.2 disabled`
|===
